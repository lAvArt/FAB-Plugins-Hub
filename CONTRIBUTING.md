# Contributing Guidelines

Thank you for your interest in contributing to the FAB Plugins Hub! This document provides guidelines for contributing to this repository.

## What You Can Contribute

While plugin source code is maintained privately, you can contribute to this public repository in several ways:

### Documentation
- Improve existing documentation
- Add tutorials and guides
- Translate documentation
- Fix typos and formatting issues
- Add examples and use cases

### Issue Reporting
- Report bugs with detailed reproduction steps
- Suggest new features
- Report compatibility issues
- Share workarounds and solutions

### Community Support
- Help answer questions in issues
- Share your experiences and best practices
- Review and improve documentation

## How to Contribute

### Reporting Issues

When reporting bugs or issues:

1. **Search Existing Issues**: Check if the issue has already been reported
2. **Use the Template**: Fill out the appropriate issue template completely
3. **Provide Details**: Include:
   - Unreal Engine version
   - Plugin version
   - Operating system
   - Steps to reproduce
   - Expected vs. actual behavior
   - Screenshots or logs (if applicable)

### Suggesting Features

When suggesting new features:

1. **Check the Roadmap**: See if it's already planned
2. **Use the Feature Request Template**: Provide detailed information
3. **Explain the Use Case**: Help us understand why this feature is needed
4. **Be Specific**: Clear descriptions help us evaluate and implement features

### Contributing Documentation

To contribute documentation:

1. **Fork the Repository**: Create your own fork
2. **Create a Branch**: Use a descriptive branch name (e.g., `docs/improve-installation-guide`)
3. **Make Your Changes**: Edit or add documentation files
4. **Follow the Style**: Match the existing documentation style and format
5. **Test Links**: Ensure all links work correctly
6. **Submit a Pull Request**: Describe your changes clearly

### Pull Request Guidelines

When submitting a pull request:

- **Clear Title**: Use a descriptive title
- **Description**: Explain what changes you made and why
- **One Topic**: Keep PRs focused on a single topic
- **Follow Format**: Match the existing documentation style
- **Test Locally**: Verify all links and formatting work

## Documentation Style Guide

### Formatting
- Use Markdown for all documentation
- Use clear, concise language
- Include code examples where appropriate
- Use proper heading hierarchy (H1 for title, H2 for main sections, etc.)

### Tone
- Be friendly and welcoming
- Use clear, simple language
- Avoid jargon when possible
- Be respectful and inclusive

### Structure
- Start with a brief overview
- Use bullet points for lists
- Include examples where helpful
- Provide links to related documentation

## Code of Conduct

### Our Pledge

We are committed to providing a welcoming and inclusive experience for everyone. We expect all participants to:

- Be respectful and considerate
- Accept constructive criticism gracefully
- Focus on what's best for the community
- Show empathy towards other community members

### Unacceptable Behavior

The following behaviors are not acceptable:

- Harassment or discrimination of any kind
- Trolling or insulting comments
- Personal or political attacks
- Publishing others' private information
- Any conduct inappropriate in a professional setting

### Enforcement

Violations of the code of conduct may result in:
- Warning
- Temporary ban
- Permanent ban

Report violations by creating a private issue or contacting the maintainers.

## Questions?

If you have questions about contributing:
- Review the [FAQ](docs/faq.md)
- Check existing [Issues](https://github.com/lAvArt/FAB-Plugins-Hub/issues)
- Create a new issue with your question

## Recognition

Contributors will be recognized in:
- Git commit history
- Release notes (for significant contributions)
- This documentation

Thank you for helping make FAB Plugins Hub better!
